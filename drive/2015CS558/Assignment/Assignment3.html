Routing Problem

In this problem you have to design a network of 4 nodes (A, B, C and D). Nodes are connected as A to B, C to B and B to D. The Physical connections between the nodes are through sockets. So if node A wants to communicate with B, it can direct connect with B because there exist a direct physical link between them. But if A wants to connect with C then it is possible only via node B. So if a wants to send a line of text to C then it will first send to B then B forward it to C. Now you have to implement this scenario. Here either of the nodes can send the text to any other node, and all the nodes (destination or mediator nodes) will display the text on its terminal.

Note: Here user will send the text by typing on the terminal of the sender node.

In this program you will implement a client and a server. The server will start in the passive mode listening on a specified port for a transmission from a client. The client will start and connect with the server on the given IP address and port number. These values will be entered via the command line. The client will send the server a string. If the string contains anything but numbers, the server will respond with "Sorry, cannot compute!" and exit. If the string contains only numbers, server will add all the digits of the strings and will return the sum as string and recursively do this until the response is a single digit (For more clarification check the example below) and exit.


Example 1:
pc1> client 172.16.25.81 90000
Enter string: 123456
Server response: 21
Server response: 3
pc1>

Multi Threaded Web Server

Currently, the web server handles only one HTTP request at a time. Implement a multithreaded server that is capable of serving multiple requests simultaneously. Using threading, first create a main thread in which your modified server listens for clients at a fixed port. When it receives a TCP connection request from a client, it will set up the TCP connection through another port and services the client request in a separate thread. There will be a separate TCP connection in a separate thread for each request/response pair.

Bulletin Board

The main of this assignment is to design a server and client for bulletin board application using C sockets. Use of already existing linux commands (grep, sort, cat etc.) for server management activities are encouraged. However to use those commands one can use "exec" system calls. Following functionalities are required to be addressed.

 

Tasks of a Server:

1. User Authentication.

2. Archiving time stamped user message to a (CSV/TSV/TXT) file.

3. Based on client request display the entire message board (Chronologically).

4. Based on client request display the filtered (Time/user) message board.

5. New user creation.

6. Delete an existing user.

7. Synchronization mechanism.

A. (Additional) Admin/privileged user support.

Tasks of a Client:

1. Secure (encrypted) transmission of user ID and password.

2. Client support for different Server options. like

a. Write a Message.

b. Delete an existing message.

c. Filtered display of messages.

d. Entire bulletin board view.

e. Password change.

f. Delete account. etc.

Segmented file transfer / Multi source File-transfer:

Consider the following scenario. A HD quality movie file is stored in multiple PCs. You want to download the movie as quickly as possible. Implement a downloader client and server using C sockets. For the sake of simplicity you can consider the following assumptions. a. You already have the IP addresses of the different source PCs, which contain the particular file.

Tasks of Server:

a. Server provides the segment information of the source file.

b. Upon receiving a request from a client, server provides the particular file to the user.

Tasks of Client:

a. Client collects the segment information from different servers.

b. Decides upon download segment and source mapping.

c. Places a request to the server.

d. Upon completion of download, concatenates different segments.
