1.

Download Center

You are asked to develop a client-server download center. On your server, there are four resources that can be downloaded: iTune , ZoneAlarm, WinRar and Audacity. (In your implementation, these resources can be represented as strings.)

The server should be always up. In order to download a resource, a client makes connection request to the server. The server sends a response to the client telling the client how to proceed. The server sends the client the terms of reference to be agreed before showing the client the resources. In order to proceed, the client has to agree the terms of reference. After that, the server sends the client the list of resources it has. The client selects a resource; the server sends the resource (i.e. the corresponding string) to the client. The client can download more resources if he/she wishes. To disconnect from the server, the client sends the message ?bye? to the server.

Here is an example of a typical exchange between the client and the server:

Server says: Here are the terms of reference. Do you accept? yes or no

Client: yes

Server says: 1. iTune 2. ZoneAlarm 3. WinRar 4. Audacity Select a resource for downloading

Client: 3

Server says: You are downloading WinRar.zip ...

Client: <CR>

Server says: Enter "bye" to exist, and "enter" to continue. Client: bye

Your application needs to include the different alternatives; for example, the client may not agree with the terms of references. The application needs to detect bad client entries, e.g. entering invalid selection for downloading.

DoS on Web Server

A. In this assignment you are required to launch a Denial of Service (DoS) attack on the web server you have already developed in Problem 2. Write a client application which can launch a denial of service attack on that particular server.

B. Web servers are configured in a way, so that it can handle basic DoS attack. One simple scheme is, setting up a threshold which determines the maximum allowable number of connection from a single IP. If a single client wants to open more than threshold number of connection, putting it in a blacklist. Modify your web server with any basic security measure for preventing DoS attack.

SSH

Client wants to execute some shell commands on the server, so ensuring that server is operating under a user account, write a server and client to allow the client to execute the command with two command line arguments:

e.g: $ ./a.out server 'shell-command'

After connecting to the server, the client sends the given command to the server, and then closes its writing half of the socket, so that the server sees end-of-file. The server should handle each incoming connection in a separate child process (i.e., a concurrent design). For each incoming connection, the server should read the command from the socket (until end-of-file), and then exec a shell to perform the command.

5.

Peer2Peer System

The objective of this assignment is to develop a peer to peer system. There will be following two different types of elements in this system.

Hub:

  • Admission control: Allowing any user to connect to the system based on valid authentication mechanism
  • Content Organization: Files shared by users are hashed by the clients/peers. Hub collects this information.
  • Content Searching: Based on the search query fetches the peer list and replies back to the client
Client/Peer:
  • Authentication: Participates in user authentication
  • Content Organization: Hashing of the shared files.
  • Content Searching: Queries the hub for the specific content
  • Downloading from peer: Upon receiving the peer listing the client uses multi-source file transfer to download the content
